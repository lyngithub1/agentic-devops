✓ Key Architecture Capabilities

✓ AI Governance Principles

1
Citations Required: Every AI-generated narrative must include source references. Output rejected if citations missing.
2
Human-in-the-Loop: AI proposes, humans decide. No auto-finalized approvals. Decision objects require explicit approval action.
3
Evidence Packs: Every opportunity includes evidence bullets, source quotes, and traceable evidenceRefs[] arrays.
4
Correlation IDs: All API responses include correlationId for end-to-end request tracing.
5
NL2SQL Auditability: Fabric Data Agent shows generated SQL and execution steps alongside every answer.

🔧 Technology Choices

A
Azure OpenAI GPT-4o: Foundation model for narrative generation, opportunity scoring, decision recommendations, and initiative planning.
B
Azure AI Search: Hybrid vector + keyword search (1536-dim embeddings via ada-002). RAG pipeline for unstructured data queries.
C
Microsoft Fabric: Data Agent with NL2SQL for structured analytics. OpenAI Assistants protocol for programmatic access. Mock-first, live-ready.
D
Zod Schemas: Runtime validation for all domain objects. TypeScript strict mode. Type-safe API contracts.
E
Managed Identity: DefaultAzureCredential for OpenAI. Key Vault for secrets. No credentials in code.
