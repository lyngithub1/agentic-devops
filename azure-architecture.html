✓ Key Architecture Capabilities

✓ Pipeline Governance Principles

1
Human-in-the-loop gates: Approvals between phases. The orchestrator pauses at each gate until a person confirms.
2
Grounded IQ: Every agent draws on Work IQ, Fabric IQ, or Foundry IQ — no ungrounded claims.
3
Orchestrated, not autonomous: The Magentic orchestrator keeps the big picture, plans, and sequences all 24 sub-agents.
4
Best-practice by design: The Developer phase applies Azure Well-Architected guidance before any code is generated.
5
Closed feedback loop: Operator telemetry and feedback are emitted as new Signals that seed the next round of Ideas.

🔧 Technology Choices

A
Azure AI Foundry: Agent Service hosts the Magentic orchestrator + 24 sub-agents; gpt-4.1 and gpt-5.4 model deployments.
B
API Management (AI Gateway): Token budgets, semantic cache, content-safety policy, and load-balancing across both models.
C
Microsoft Fabric: Fabric IQ business data via OneLake/Lakehouse + Data Agent; Real-Time Intelligence and Power BI for Operator.
D
Work IQ (Microsoft Graph): People, work items and context; GitHub for real repos and pull requests.
E
Managed identity: Entra ID + per-agent Entra Agent ID; secrets in Key Vault. No model keys ever reach the browser.
